## Enabling Cowork: The Frontier Path

**Post 2 of 14**

## Introduction

Cowork is not a feature you flip on like Loop or Designer. It is a Frontier preview feature, which means access is gated by a tenant-level opt-in plus an admin enrolment of their own account. If Cowork is missing from your Microsoft 365 Copilot view, the most common cause is step zero.

## The Four-Step Path to Access

**Prerequisites.**
- An active Microsoft 365 Copilot license on the user account.
- A tenant enrolled in the Frontier preview program.
- A modern browser (Edge or Chrome) or the Microsoft 365 Copilot desktop app for Windows or Mac.

**Step 1 — Enroll the tenant in Frontier.** In the Microsoft 365 Admin Center, go to **Copilot → Settings → Frontier** and enrol your tenant. Frontier previews are subject to your existing customer agreement preview terms.

**Step 2 — Enroll the admin account.** This trips up most teams. If Cowork does not appear in **Agent management**, it is because the admin's own account is not enrolled in Frontier. Enroll the account, sign out, and back in.

**Step 3 — Make Cowork available to users.** In **Agent management**, manage which users see Cowork. Pilot rings first, broad rollout later.

**Step 4 — Open Cowork.** Users navigate to `m365.cloud.microsoft`, the desktop app, or the agent surface in Outlook or Teams, and select **Cowork** (or find it under **All agents**).

**What users see on first run.** A chat input, a side panel for files, suggested prompts (`Catch me up`, `Organize my inbox`, `Prep for a meeting`), the **Tasks** view in the navigation, and an output panel where generated files land.

**What admins should pre-decide.**
- Which security groups get Cowork.
- Whether the Anthropic subprocessor toggle is enabled (default in many regions; opt-in for EU/UK/EFTA — Post 3 covers this).
- Sensitivity labels and DLP policies that should apply before broad rollout.
- Audit/SIEM coverage for Cowork sessions.

**Disabling Cowork for specific users.** Add them to a security group configured to exclude them from the Copilot experience, or manage availability through the Copilot settings in the admin center.

## Takeaway

The single most common rollout issue is the admin enrolment step. Enroll both the tenant's and the admin's accounts in Frontier before troubleshooting anything else.
